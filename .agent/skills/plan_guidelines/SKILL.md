# 플랜 작성 가이드라인 (Plan Guidelines)

이 스킬은 사용자에게 플랜을 제시할 때 일관된 언어와 형식을 유지하기 위해 사용됩니다.

## 주요 규칙

1. **한국어 사용**: 모든 플랜과 설명은 기본적으로 한국어로 작성합니다.
2. **기술 용어 병기**: 기술적인 용어의 경우, 혼동을 방지하기 위해 소괄호`()`를 사용하여 영어 표현을 동시에 표기합니다.
   - 예: "인증 시스템(Authentication System)을 구현합니다."
3. **가독성**: 사용자가 이해하기 쉽도록 명확하고 간결한 문장을 사용합니다.

## 예시

```markdown
# 사용자 프로필 기능 구현 (Implementing User Profile Feature)

1. 데이터베이스 스키마(Database Schema) 수정
2. API 엔드포인트(API Endpoint) 생성
3. 프론트엔드 컴포넌트(Frontend Component) 개발
```
